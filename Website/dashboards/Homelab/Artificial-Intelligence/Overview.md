# Artificial Intelligence

This dashboard is for local-first AI notes, tools, experiments, and setup
recipes.

AI can be useful in a resilient homelab, but only if it still works when the
internet is unavailable. A local model that can summarize notes, explain logs,
draft scripts, or search your own documentation is a lot more useful than a
cloud demo that turns into a paperweight the moment the router starts sulking.

## What Belongs Here

Use this dashboard for:

- Local model setup notes
- Ollama, LM Studio, and OpenAI-compatible endpoint experiments
- Prompt patterns that are actually worth reusing
- GPU, CPU, RAM, and storage notes for model hosting
- Offline documentation workflows
- AI-assisted maintenance scripts

## Local-First AI Goals

The practical target is not "AI everywhere." The practical target is useful AI
where it reduces friction:

| Goal | Why It Matters |
| :--- | :--- |
| Local inference | Keeps basic AI tools available without cloud access. |
| Private notes | Avoids sending sensitive homelab or personal data to remote services. |
| Repeatable setup | Makes rebuilds less painful when a machine gets replaced. |
| Small models first | Saves power, storage, and patience. |

## Good Starting Points

Start simple:

1. Pick one local runtime.
2. Pick one model that fits your hardware.
3. Document the exact install steps.
4. Save known-good prompts and commands.
5. Only then start adding the fancy stuff.

The fancy stuff is fun. The working stuff is useful.
