# Self-Hosting Dashboard

This dashboard is the Homelab command post for services you run yourself.

Use it for notes about local apps, reverse proxies, backups, storage, container
stacks, update habits, and the tiny operational rituals that keep a home server
from becoming a mysterious blinking box in the corner.

## Good Uses

- Service inventory and port notes.
- Docker, Podman, or VM maintenance checklists.
- Backup and restore runbooks.
- Reverse proxy and DNS notes.
- Offline installer and container image reminders.
- Recovery steps for when a service says "it worked yesterday" with a straight
  face.

## Practical Self-Hosting Notes

Self-hosting is not just about replacing cloud apps. It is about knowing where
your data lives, how your services start, and how to recover when the internet
is unavailable or a vendor changes the rules.

Keep the important stuff boring:

- Document how each service is installed.
- Keep backups somewhere boring and testable.
- Write down how to restore, not just how to deploy.
- Prefer simple service stacks you can explain when tired.

## Dashboard Notes

Dashboards are public local content. They are not encrypted with `git-crypt`.
Put private notes under Profiles instead.
