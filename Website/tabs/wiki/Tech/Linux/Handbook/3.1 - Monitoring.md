# Monitoring Linux

Monitoring is how you answer the question:

```text
What is my system doing right now?
```

It is the difference between "my laptop feels weird" and "Firefox is eating 6
GB of RAM, Docker is hammering the disk, and something is talking to the
network every five seconds." The first one is vibes. The second one is
evidence.

Good monitoring does not have to be complicated. You do not need a giant
enterprise dashboard to understand one Linux machine. Start with simple tools,
learn what normal looks like, and then you will notice when something is not
normal.

## What Should You Monitor?

For a normal Linux desktop or workstation, focus on these areas:

- CPU usage
- Memory usage
- Disk usage
- Disk activity
- Network activity
- Running processes
- Open ports
- Startup services
- Temperatures and fan behavior
- Battery health on laptops

That sounds like a lot, but most of it can be checked with a few commands.

## The First Rule: Learn Normal

Monitoring only becomes useful when you know what your system normally looks
like.

Check your machine when it is idle. Check it while browsing. Check it while
installing updates. Check it while gaming, compiling, copying files, or running
containers.

You are building a mental baseline.

If your machine normally idles at 2 percent CPU and suddenly idles at 35
percent, that is a clue. If your network is usually quiet and suddenly a random
process is constantly sending data, that is also a clue. If your laptop fan
sounds like it is preparing for orbit, that clue is less subtle.

## Process Monitoring

Processes are running programs. When your system feels slow, start here.

### top

`top` is installed almost everywhere:

```bash
top
```

It shows CPU usage, memory usage, load average, and running processes.

Useful keys inside `top`:

- `q` quits
- `P` sorts by CPU
- `M` sorts by memory
- `k` kills a process

`top` is not pretty, but it works. It is the folding chair of Linux monitoring.

### htop

`htop` is friendlier:

```bash
sudo apt install htop
htop
```

On Arch:

```bash
sudo pacman -S htop
htop
```

On Fedora:

```bash
sudo dnf install htop
htop
```

`htop` gives you colors, scrolling, process trees, and easier sorting. If you
are new to Linux monitoring, install it.

### btop

`btop` is a more modern terminal monitor:

```bash
btop
```

It gives you a clean view of CPU, memory, disks, network, and processes. It is
one of the best "single screen" tools for checking system health.

Install it with your distro package manager:

```bash
sudo apt install btop
sudo pacman -S btop
sudo dnf install btop
```

Use whichever command matches your distro.

## Memory Monitoring

To check memory:

```bash
free -h
```

Example output:

```text
               total        used        free      shared  buff/cache   available
Mem:            15Gi       4.2Gi       6.1Gi       650Mi       5.4Gi        10Gi
Swap:          2.0Gi          0B       2.0Gi
```

The important column is usually `available`, not `free`.

Linux uses spare memory for cache because unused RAM is wasted RAM. New users
often see "used memory" and panic. Linux is not necessarily hoarding your RAM
like a dragon. It is probably caching files to make things faster.

## Disk Space Monitoring

To check disk space:

```bash
df -h
```

To check the size of folders in the current directory:

```bash
du -h --max-depth=1
```

To sort folder sizes:

```bash
du -h --max-depth=1 | sort -h
```

If your disk fills up, Linux can get weird fast. Logs may stop writing, apps may
fail, package updates may break, and your desktop may start acting like it is
personally offended.

Keep free space available.

## Disk Activity

If the system feels frozen but CPU usage is low, disk activity may be the
problem.

Install `iotop`:

```bash
sudo apt install iotop
```

Then run:

```bash
sudo iotop
```

This shows which processes are reading and writing to disk.

Another useful tool is `iostat`, usually from the `sysstat` package:

```bash
sudo apt install sysstat
iostat -xz 1
```

If a disk is constantly busy, check backups, indexing services, containers,
browser caches, virtual machines, and anything syncing files.

## Network Monitoring

Network monitoring answers:

```text
Who is my computer talking to?
```

That is important for privacy, security, troubleshooting, and general curiosity.
Curiosity is healthy. Paranoia with commands is basically system administration.

### ss

To show listening ports:

```bash
ss -tulpn
```

You may need root to see process names:

```bash
sudo ss -tulpn
```

Useful flags:

- `t` shows TCP
- `u` shows UDP
- `l` shows listening sockets
- `p` shows processes
- `n` avoids slow DNS/service lookups

If you see a service listening on a port, ask whether you expected it.

### ip

To see network interfaces:

```bash
ip addr
```

To see routes:

```bash
ip route
```

This helps you understand your local IP, gateway, VPN routes, and whether the
machine is using the network you think it is using.

### nload

`nload` gives a simple live view of network traffic:

```bash
sudo apt install nload
nload
```

It is useful when you want to know whether a machine is currently uploading or
downloading data.

### iftop

`iftop` shows which remote addresses are using bandwidth:

```bash
sudo apt install iftop
sudo iftop
```

This is helpful when the network is busy and you want to know where the traffic
is going.

### Little Snitch For Linux

A graphical network monitor/firewall can also be useful.

Little Snitch for Linux is one option for observing and controlling outbound
and inbound traffic:

https://obdev.at/products/littlesnitch-linux/index.html

The idea is simple: instead of only seeing traffic after the fact, you can be
asked when an application tries to connect. That can teach you a lot about what
desktop apps are doing in the background.

Use tools like this carefully. If you block everything because every popup
looks scary, your system may become secure in the same way a brick is secure:
nothing works, but technically nothing is leaking either.

## Service Monitoring

Most modern Linux distros use `systemd`.

To list failed services:

```bash
systemctl --failed
```

To check a service:

```bash
systemctl status ssh
```

To list enabled services:

```bash
systemctl list-unit-files --type=service --state=enabled
```

To list running services:

```bash
systemctl list-units --type=service --state=running
```

Services are where Linux hides a lot of important behavior. If something starts
automatically, opens a port, syncs files, mounts drives, or runs on a timer,
there is a good chance `systemd` knows about it.

## Startup And Timers

To see timers:

```bash
systemctl list-timers
```

Timers are the `systemd` version of scheduled jobs. They can run updates,
cleanup tasks, backups, indexing, and other background work.

Traditional cron jobs may also exist:

```bash
crontab -l
sudo crontab -l
```

Also check:

```bash
ls /etc/cron.*
```

If something keeps happening "by itself," it is often a service, timer, cron
job, desktop autostart entry, or script you forgot you created during an
ambitious evening.

## Temperature And Hardware Sensors

Install sensors:

```bash
sudo apt install lm-sensors
sudo sensors-detect
sensors
```

On Arch:

```bash
sudo pacman -S lm_sensors
sudo sensors-detect
sensors
```

On Fedora:

```bash
sudo dnf install lm_sensors
sudo sensors-detect
sensors
```

Temperature monitoring matters because heat causes throttling, instability, fan
noise, and sometimes mysterious shutdowns.

If a machine is always hot:

- Clean dust from vents.
- Check fan behavior.
- Check CPU load.
- Check GPU load.
- Replace thermal paste if the machine is old and you are comfortable doing it.
- Make sure laptops have airflow.

The laptop sitting on a blanket is not "cozy." It is suffocating.

## Battery Monitoring

For laptops:

```bash
upower -e
```

Then inspect the battery path:

```bash
upower -i /org/freedesktop/UPower/devices/battery_BAT0
```

You can also check:

```bash
cat /sys/class/power_supply/BAT0/capacity
cat /sys/class/power_supply/BAT0/status
```

Battery health matters for portable systems. A failing battery can cause sudden
shutdowns, weird sleep behavior, or performance issues.

## A Simple Daily Monitoring Checklist

When a Linux machine feels wrong, check:

1. `btop` or `htop`
2. `df -h`
3. `free -h`
4. `systemctl --failed`
5. `journalctl -p warning -b`
6. `ss -tulpn`
7. `sensors`

That gives you a fast overview of processes, disk, memory, failed services,
recent warnings, open ports, and temperatures.

## Final Thoughts

Monitoring is not about staring at graphs all day. It is about knowing how to
look when something feels off.

Learn the normal shape of your system. Learn which services should be running.
Learn which ports should be open. Learn what idle CPU, memory, disk, and
network activity look like.

Then when something changes, you will notice.
