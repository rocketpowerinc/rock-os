# CLI Tools Bootstrap

A fresh Linux install without a decent CLI toolkit feels like walking into a
garage with one screwdriver and a suspiciously sticky flashlight.

This page is the quick bootstrap for the core command-line tools I like to have
available early. Some are required for normal Git work, some are useful for
automation, and some just make the terminal feel less like a punishment from
1997.

## Why This Matters Offline

In a grid-down or offline scenario, the terminal becomes less of a preference
and more of a lifeline. You may not have app stores, cloud dashboards, search
engines, package mirrors, or that one forum post from 2014 where some hero
fixed your exact problem with a single command.

These tools help you keep working locally:

- `git` lets you preserve notes, configs, scripts, and recovery docs with real
  history instead of a folder full of files named `final-final-USE-THIS-one`.
- `jq`, `curl`, and `wget` help inspect local APIs, saved JSON, exported data,
  and offline mirrors.
- `tmux` keeps long-running jobs alive if your terminal disconnects or your
  laptop decides now is a great time to be dramatic.
- `make` gives you repeatable commands so recovery steps are not trapped in
  memory, caffeine, and vibes.
- `just` gives you a friendlier command runner for project tasks, especially
  when you want `just backup`, `just rebuild`, or `just fix-the-thing` instead
  of a ceremonial wall of shell incantations.
- `go` lets you rebuild local tools from source when binaries are missing,
  stale, or built for the wrong machine.
- `bat` and `glow` make local documentation readable when the internet is gone
  and your notes are suddenly the library.

The whole point is boring resilience. Boring is good. Boring means the tool
worked and nobody had to invent a new swear word.

## What This Installs

| Tool | Why It Matters |
| --- | --- |
| `git` | Repository work. No Git, no party. |
| `gh` | GitHub CLI for releases, issues, PRs, and auth. |
| `jq` | JSON processing for scripts and API output. |
| `make` | Common build and automation wrapper. |
| `just` | Modern command runner for repeatable project tasks. |
| `go` | Needed for building or running Go projects from source. |
| `bat` | Better `cat` with syntax highlighting. Very civilized. |
| `tmux` | Terminal sessions that survive disconnects and chaos. |
| `curl` / `wget` | Downloading files, APIs, install scripts, and trouble. |
| `glow` | Renders Markdown in the terminal. Fancy, but useful fancy. |
| `gum` | Nice interactive shell prompts and script UI. |
| `powershell` | Optional, useful if you work with `.ps1` scripts cross-platform. |

## Ubuntu / Debian

Start with the basics:

```bash
sudo apt update
sudo apt install -y git gh jq make just bat tmux curl wget golang-go
```

`glow` and `gum` may not be available in older default repositories. The clean
way is to use Charm's official repo:

```bash
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://repo.charm.sh/apt/gpg.key | sudo gpg --dearmor -o /etc/apt/keyrings/charm.gpg
echo "deb [signed-by=/etc/apt/keyrings/charm.gpg] https://repo.charm.sh/apt/ * *" | sudo tee /etc/apt/sources.list.d/charm.list
sudo apt update
sudo apt install -y glow gum
```

PowerShell 7 is optional. Install it if you actually need `.ps1` support:

```bash
wget -q https://packages.microsoft.com/config/ubuntu/22.04/packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
sudo apt update
sudo apt install -y powershell
```

> [!NOTE]
> Debian users may need to use the Microsoft package feed that matches their
> Debian version instead of the Ubuntu package shown above. Cross-distro package
> feeds are powerful, but so is a circular saw.

## Fedora

Fedora usually gives you nice modern packages without a wrestling match:

```bash
sudo dnf install -y git gh jq make just bat tmux curl wget golang glow gum
```

PowerShell is optional:

```bash
sudo dnf install -y powershell
```

If `powershell` is not available, enable Microsoft's Fedora repo first or skip
it until you actually need it.

## Arch Linux

Arch keeps this refreshingly direct:

```bash
sudo pacman -Syu git github-cli jq make just bat tmux curl wget go glow gum
```

PowerShell is usually installed from the AUR:

```bash
paru -S powershell-bin
```

If you use `yay` instead of `paru`, swap the command:

```bash
yay -S powershell-bin
```

If you do not use an AUR helper yet, skip PowerShell for now. Get the system
stable first, then decide how many moving parts you want in your backpack.

## NixOS

For a temporary shell, use:

```bash
nix shell nixpkgs#git nixpkgs#gh nixpkgs#jq nixpkgs#gnumake nixpkgs#just nixpkgs#bat nixpkgs#tmux nixpkgs#curl nixpkgs#wget nixpkgs#go nixpkgs#glow nixpkgs#gum
```

For a real NixOS install, add the packages to `environment.systemPackages` in
`/etc/nixos/configuration.nix`:

```nix
environment.systemPackages = with pkgs; [
  git
  gh
  jq
  gnumake
  just
  bat
  tmux
  curl
  wget
  go
  glow
  gum
  powershell
];
```

Then rebuild:

```bash
sudo nixos-rebuild switch
```

NixOS is the place where "installed" means "declared into existence like a
wizard with a clipboard." Strange at first, excellent once it clicks.

## Go User Path

Go installs command-line tools into `~/go/bin` by default. Add it to your shell
path so anything installed with `go install` can actually run:

```bash
echo 'export PATH="$HOME/go/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

If you use Zsh:

```bash
echo 'export PATH="$HOME/go/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

## Quick Sanity Check

After installing, check the important pieces:

```bash
git --version
gh --version
jq --version
make --version
just --version
go version
bat --version
tmux -V
curl --version
wget --version
glow --version
gum --version
```

PowerShell, if installed:

```bash
pwsh --version
```
